1. 변형과 왜곡

사다리꼴처럼 보이는 형태는 단순 크기 조절이 아니라 원근감이 들어간 왜곡으로 이해해야 합니다.

Free Transform

사다리꼴 만드는 순서

  1. 오브젝트를 선택하고 Free Transform을 연다.
  2. 옵션에서 Perspective 기능을 사용한다.
  3. 모서리를 드래그해 위아래 폭 차이를 만든다.
  4. 직사각형이 사다리꼴처럼 바뀌면 왜곡과 원근감이 적용된 것이다.

핵심 해석

  • 이 작업은 단순 변형이 아니라 왜곡을 이용한 형태 조정이다.
  • Perspective는 화면이 멀어지는 느낌, 즉 원근감을 만든다.
  • 텍스트에도 적용할 수 있지만, 수정 단계인지 최종 단계인지 먼저 판단해야 한다.
정리: 수업 메모의 흐름은 사다리꼴 → Free Transform → Perspective → 왜곡입니다. 즉 사다리꼴 모양이 보이면 어떤 메뉴를 눌렀는지보다, 왜 그런 모양이 되었는지까지 같이 기억해야 합니다.

2. 텍스트 작성과 전환

문자를 입력하는 상태와 오브젝트처럼 선택하는 상태를 구분해야 이후 작업이 꼬이지 않습니다.

Type 메뉴

문자 확정

  • 텍스트 입력 후 Ctrl + Enter로 문자 입력을 확정한다.
  • 확정 전에는 입력 중이고, 확정 후에는 오브젝트처럼 선택 가능하다.

대문자 변경

  • Type > Change Case 메뉴를 사용한다.
  • 영문 제목이나 강조 텍스트를 빠르게 정리할 때 자주 쓴다.

텍스트 종류 전환

  • 클릭해서 만들면 Point Text다.
  • 드래그해서 만들면 Area Text다.
  • Type > Convert to Point Text
  • Type > Convert to Area Text

헷갈리기 쉬운 포인트

클릭 → Point Text / 드래그 → Area Text입니다.

순서를 거꾸로 기억하면 나중에 텍스트 상자 구조를 설명할 때 계속 헷갈립니다.

빠른 확인 질문

  • 지금 선택한 텍스트는 입력 중인가?
  • 클릭형 텍스트인가, 영역형 텍스트인가?
  • 문자 상태로 수정할 단계인가, 윤곽선으로 넘길 단계인가?

3. 도형화: 윤곽선 만들기

이번 메모에서 가장 중요한 파트입니다. 별표가 많은 이유가 있는 내용입니다.

중요 ★★★★★

Create Outlines

  • Type > Create Outlines
  • 단축키: Ctrl + Shift + O
  • 글자를 더 이상 문자로 보지 않고 패스 형태로 바꾼다.
  • 수업에서는 보통 글자를 '깬다'고 표현한다.

왜 중요한가

  • 문자 상태로는 자유로운 패스 편집에 한계가 있다.
  • 왜곡, 합치기, 로고 작업은 도형 상태가 더 유리하다.
  • 출력 환경에 폰트가 없어도 글자 모양을 유지하기 쉽다.

반드시 기억할 문장

글자는 글자인 상태로는 자유롭게 형태 편집을 하기 어렵다.

형태처럼 다루려면 윤곽선 만들기를 해야 한다.

Fill 컬러가 채워져 있다고 해서 선인 것은 아니다. 채워진 면과 실제 Stroke를 구분해야 한다.

교재 메모

  • 264p 내용을 다시 확인하기
  • 문자 상태와 윤곽선 상태 비교해서 보기

4. 복사와 후처리 도구

도형화 이후에는 복제, 잠금, 제자리 붙여넣기, 합치기 도구가 자주 이어집니다.

실무 연결

자주 쓰는 단축키

복사 Ctrl + C
잠금 Ctrl + 2
제자리 복제 Ctrl + F
잠금 해제 Ctrl + Alt + 2

함께 기억할 도구

1
Pathfinder는 여러 도형을 합치거나 빼는 데 사용한다.
2
Shape Builder는 드래그 기반으로 직관적으로 합치거나 삭제할 수 있다.
3
Offset Path는 기존 패스를 기준으로 안쪽 또는 바깥쪽에 새 경로를 만든다.
윤곽선으로 만든 텍스트는 이후 로고, 배지, 스티커형 타이포, 장식 그래픽 작업으로 자주 이어집니다.

4. 글자를 도형처럼 편집하기 위해 윤곽선으로 바꾸는 단축키는?
